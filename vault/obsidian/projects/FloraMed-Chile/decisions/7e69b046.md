# ADR: Pl@ntNet API service isolated in dedicated module

**Status**: proposed
**ID**: 7e69b046
**Proyecto**: [[projects/FloraMed-Chile/README|FloraMed-Chile]]

## Description

Created plantas/services/plantnet_service.py with a single-responsibility function identificar_planta() that handles image upload, multipart encoding, API communication, error mapping, and DEBUG-mode mock fallback. The service raises typed PlantNetError so views can render user-facing messages without leaking internals.

## Rationale

1) Inline API calls in views (violates SRP, hard to test). 2) Django model method (wrong layer). 3) Celery task (overkill for synchronous identify).
